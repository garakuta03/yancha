## 1. YouTubeポリシー

### 1.1 YPP「inauthentic content（量産型コンテンツ）」ポリシー

- 2025-07-15にYPPの「繰り返しの多いコンテンツ」が「inauthentic content」へ改称。
  「repetitive or mass-produced を含むことの明確化」であり新規禁止事項の追加ではない
  （[公式ヘルプ](https://support.google.com/youtube/answer/1311392) /
  [Social Media Today](https://www.socialmediatoday.com/news/youtube-clarifies-monetization-update-inauthentic-repeated-content/752892/)）。
- **収益化NGの公式例**: 動画間の差分が最小限の類似/反復コンテンツ、画像スライドショー・
  物語性のないスクロールテキスト、**「汎用テンプレートで量産の印象を与えるAI生成コンテンツ」**。
- **収益化OKの公式例**: イントロ/アウトロは同一でも本編が異なる、形式は似ていても各動画に
  固有のストーリー・焦点・コンセプトがあるもの。
- YouTube公式（Creator Liaison）は「**AIの禁止ではない**。AIをツールとして使い独自の
  物語・視点を加えるコンテンツは収益化可能」と明言。
- **違反判定はチャンネル単位**（1本でなく全体が収益化剥奪対象）。
- アンビエント/lofi/瞑想系はYPP審査却下率が高いジャンル群とされる（業界解説。
  [Dynamoi](https://dynamoi.com/learn/youtube-music-promotion/ambient-music-channel-monetization-rules)）。
  日本語圏でも自動生成BGMチャンネルの収益化停止報告が2026年に増加
  （[note当事者報告](https://note.com/grand_yarrow659/n/n3ee0c3e7908c) 等）。
- 通過チャンネルの共通項（community定説）: 自作音源の証明可能性、動画ごとの映像・音楽の固有性、
  チャンネルとしてのキュレーション付加価値。
- ⚠️ 未検証情報（信頼度低）: 「三振制」「2026年1月に全AI開示義務化」「完全AI音源はContent ID対象外」
  — 公式一次ソースで確認できず。

### 1.2 AI開示義務（2026年5〜6月に大改訂: 「GenAIコンテンツの開示」）

- 現行ポリシーは「Disclosing use of GenAI content」（[公式・生HTMLで原文確認済み](https://support.google.com/youtube/answer/14328491)）。
  開示必須: 実在人物の言動の捏造、実在の出来事・場所の映像の改変、
  **「実際には起きていない場面が現実のように見えるコンテンツの生成」**。
- **「AI generated music（AI生成の音楽）」は開示必須例の筆頭に明記**（2024年の導入当初から一貫。
  「BGMだから不要」は誤り）。
- 開示不要の公式例: 非現実的コンテンツ（幻想世界・フルアニメーション）、軽微な編集（色調整・背景ぼかし・
  台本/サムネ生成等の制作補助・**自分の声のクローン**）。
- **2026-05-27から自動検出・自動ラベル付与が導入**: YouTube内部システムがAI生成と検出した場合・
  C2PAメタデータを含む場合は自動でラベルが付く（C2PA付き・手動レビュー後のラベルは解除不可）
  （[公式ブログ](https://blog.youtube/news-and-events/improving-ai-labels-viewers-creators/) /
  [TechCrunch](https://techcrunch.com/2026/05/27/youtube-will-now-automatically-label-ai-videos/)）。
- ラベル表示: **フォトリアルはプレーヤー上、非フォトリアル/アニメ調は説明欄内のみ**（目立たない）。
- **「開示ラベル単体は推薦・収益化に影響しない」と2024年から2026年5月まで一貫して公式明言**。
  ペナルティ（ラベル強制付与・削除・YPP停止）は**常習的な未開示側にのみ**存在。
- ただし別系統のリスクとして、2026年1月にCEOが「AI slop」対策を宣言し16チャンネル
  （計3,500万登録）を停止。**顔出しなしのアンビエント系が取り締まりの巻き添えになっている**との報道あり
  （[The Next Web](https://thenextweb.com/news/youtube-ai-slop-crackdown-faceless-creators-collateral-damage)）。
  根拠は開示義務でなくスパム/inauthenticポリシー。
- 実務結論: **開示は常にON（コスト実質ゼロ・非フォトリアルなら表示も目立たない）**。
  映像は「明らかに非実写（アニメ調・抽象・プロシージャル）」に寄せると、開示論点の回避と
  プレーヤー上ラベルの回避を両立できる。真のリスクは開示でなく量産型判定（§1.1）。

### 1.3 ライブ配信の仕様（2026年時点）

| 項目 | 内容 |
|---|---|
| 24/7連続配信 | 上限の公式規定なし。Lofi Girlが2年以上連続配信の実績（[Wikipedia](https://en.wikipedia.org/wiki/Lofi_Girl)） |
| アーカイブ | **12時間未満のみ自動アーカイブ。12時間超はキャプチャされない場合がある**（2025-11にもTeamYouTubeが回答。[公式](https://support.google.com/youtube/answer/6247592)） |
| YPP 4,000時間 | **公開アーカイブされたライブの視聴時間はカウントされる**（[公式](https://support.google.com/youtube/answer/72851)）。12時間超・アーカイブなし配信の計上は不透明 |
| ストリームキー | 「デフォルト配信」キーは永続。スケジュールイベント用キーはイベント限り |
| 切断猶予 | 公式値なし。community報告では30秒〜数分バッファリングで維持、同キー再接続で継続 |
| 有効化条件 | チャンネル確認（電話）＋16歳以上＋過去90日にライブ制限なし。初回有効化に待機期間あり（[公式](https://support.google.com/youtube/answer/2474026)） |
| ループ配信 | ループ自体の禁止規定なし。リスクは (a) 素材使い回しのinauthentic判定 (b) 虚偽DMCA/Content IDで配信ごと停止（Lofi Girlですら2022年に虚偽申立てで一時停止） |

**実務結論**: ライブは**12時間未満のブロックに区切って配信→公開アーカイブ化**が、
視聴時間計上・アーカイブ二重活用・切断リスクの全てで有利。

### 1.5 Content ID（オリジナルAI音楽の防衛）

- 誤クレームの構造的リスク: 同系ツールの類似AI曲を第三者が先にディストリビューター経由で
  Content ID登録→自分の動画に誤マッチ、が多発（[Undetectr](https://undetectr.com/blog/ai-music-content-id)）。
  AI音楽を使ったロイヤリティ詐欺の報道もある（[Forbes 2026-05](https://www.forbes.com/sites/virginieberger/2026/05/05/how-ai-generated-music-became-a-4-billion-fraud-machine/)）。
- 自作AI曲のContent ID登録は**期待しない**: 個人直接参加は不可能、ディストリビューターは
  AI曲の登録を拒否・厳格化する傾向（DistroKidは2026-05からAI申告欄導入、AmuseはAI曲の登録停止報道）。
- 防衛は**証跡主義＋dispute**: 生成ログ・プロンプト・ステム・編集履歴・公開前タイムスタンプの保存。
  公開前に**限定公開でContent IDスキャンを通してから公開**する運用が有効。

## 2. 音楽・環境音

### 2.1 商用生成サービス（Suno/Udio）

- **Suno**: 有料プランは生成物の権利をユーザーに譲渡・YouTube収益化可（[規約](https://suno.com/terms-of-service)）。
  ただし (a) 出力の非侵害保証なし・補償はユーザー→Suno片務、(b) **UMG・Sonyとの訴訟は係争中**
  （2026-05に対象楽曲61,026曲へ拡大・損害理論値$9B超・事実審理2027年見込み）、
  (c) Warnerとは2025-11に和解済みで現行モデルは廃止予定・DL制限導入へ。
- **Udio**: 2025-10のUMG和解後、**ダウンロード禁止・プラットフォーム内限定**となり外部利用は実質不可能。
- エンドユーザー（生成物の商用利用者）が訴えられた事例は確認されず（攻撃対象はプラットフォーム本体）。
- 評価: Sunoは「有料プラン＋即DL＋規約スナップショット保存」でフェーズ0の品質ベンチマーク用途まで。
  **主軸には据えない**。

### 2.3 プロシージャルDSP合成（環境音）— 権利リスク実質ゼロの本命

- 学習モデルも録音素材も使わない純合成は、**第三者著作権・隣接権侵害の経路が構造的に存在しない**。
  Content ID誤爆リスクも最小。
- 定番教科書: Andy Farnell『Designing Sound』(MIT Press)。**雨・風・火・水・雷のPdパッチが無償公開**
  （[aspress.co.uk/sd](https://aspress.co.uk/sd/)）。SuperCollider移植版Wikibookあり。
- 品質実勢: 風・ノイズ系・波は純合成で実用十分。雨は「中景の雨」なら商用品質
  （Audiokinetic公式チュートリアル・市販プロシージャル雨プラグインAudioRainの存在が証左）。
  焚き火の低域うねりが最難関。myNoiseは実録音主義だが合成のみのジェネレータも持つ。
- ツール（いずれも出力音声にツールのライセンスは及ばない。[GNU GPL FAQ](https://www.gnu.org/licenses/gpl-faq.en.html)）:

| ツール | ライセンス | TS親和性 | 備考 |
|---|---|---|---|
| **Elementary Audio + node-web-audio-api** | MIT | **◎ TSネイティブ** | `@elemaudio/offline-renderer`でNodeヘッドレスレンダリング公式対応 |
| SuperCollider NRT | GPLv3（サブプロセス起動なら無問題） | ○ | `-N`で実時間より速くオフラインレンダリング。batchNRT Quarkあり |
| Csound | LGPL | ○ | デフォルトが非実時間レンダラ。最も枯れている |
| Pure Data / libpd | BSD | △ | Farnellパッチの参照実装・研究用 |
| Rust fundsp | MIT/Apache-2.0 | ○（CLI化） | 性能最良。開発コスト増 |

- 注意: OfflineAudioContextは全結果をメモリに持つ（10時間48kHz2chで約13GB）→
  **10〜30分チャンクでレンダリング→ffmpeg concat**の設計が必須。
- CC0実録素材のレイヤーは「偽CC0混入」「第三者のContent ID登録による誤クレーム」の実リスクがあり、
  完全プロシージャルに劣後。

### 2.4 著作権法制（生成物の保護）

- **日本（文化庁「AIと著作権に関する考え方」2024-03、チェックリスト2024-07）**:
  - 著作物性: プロンプトのみ・短い指示では創作的寄与と認められない可能性が高い。
    試行の繰り返し＋指示修正、人間の創作的な加筆・修正部分には著作物性が認められ得る。
  - 侵害判断は類似性＋依拠性。**学習データに当該著作物が含まれれば利用者が知らなくても依拠性が推認され得る**
    →「作家名をプロンプトに入れていないから安全」は不成立。学習データが不透明なツールほど反証困難。
  - 利用者の推奨行動: 学習データ情報の事前確認、公開前の類似確認（検索等）、**生成過程の記録・保存**。
- **米国（USCO Part 2, 2025-01）**: プロンプトのみでは著作権不発生。人間の表現的関与部分は保護。
- **実務影響**: 著作物性のない音源・映像は第三者コピーを著作権で止められない（日本では民法不法行為も
  北朝鮮事件最判の枠組みで原則否定）。→ 対策は「人間の編曲・構成・ミックスを挟んで著作物性を確保」
  「動画全体（構成・映像・音の組合せ）として保護される状態にする」「チャンネルブランドで差別化」。
  **プロシージャル生成はコードとパラメータを人間が全て書くため、AI自律生成より著作物性が認められやすい。**

### 2.5 類似性チェック

| 手法 | 用途 |
|---|---|
| 限定公開アップロード→Content IDスキャン確認→公開 | 必須運用に推奨（無料） |

### 3.2 動画生成モデル

- ライセンス第一候補は **Wan 2.2（Apache 2.0）**。ただし**Macでは非実用的**
  （M1 Max実測: 2秒クリップに82分。[lilting.ch](https://lilting.ch/en/articles/ltx2-wan22-mac-local-video-gen)）。
  A2000 12GBならTI2V-5B FP8＋オフロードで5秒720p可。
- LTX-2はCommunity License（$10M未満商用可）だが**AI生成の明示表示義務＋違約金条項**あり。
  HunyuanVideoは独自条項多数（EU/UK/KR除外・香港法準拠）。
- **フェーズ0の現実解: 静止画（Apache 2.0系）＋ffmpegの微動化（zoompan/パララックス/パーティクル）**。

### 3.3 Civitai等の派生モデルの落とし穴

- ベースモデルのライセンスはマージ・LoRAで消えず継承される。多段マージは出所追跡が不可能で
  非商用素材の汚染リスク。特定作家の絵柄を学習したファインチューンは依拠性リスクがベースより構造的に高い。
- **結論: Hugging Face公式リポジトリのApache 2.0/MITベースモデルのみを使う。**

### 3.4 プロシージャル映像（AI不要・法的に最安全）

- ffmpeg内蔵ジェネレータ（gradients/mandelbrot/cellauto等）＋フィルタ連鎖、静止画＋zoompan、
  gl-transitions（MIT）。
- **three.js（MIT）/ p5.js のヘッドレスレンダリング**（Puppeteer＋フレームキャプチャ→ffmpeg）で
  シェーダ・パーティクル・フローフィールドを決定論的に任意尺レンダリング可能。
- コードが自作なら**生成物は人間の創作物として著作権を主張できる**（AI生成物の著作権不発生問題も回避）。
  TS技術方針と完全に一致。
- 注意: プロシージャルでも「反復感」はinauthentic判定リスクになるため、パラメータ空間を広く取る。

## 4. ニッチ市場（声なし×自動生成適性）

### 4.1 定番ニッチの飽和度

| ニッチ | 飽和度 | 備考 |
|---|---|---|
| 睡眠音楽/雨音 | 高（需要も最大） | RPMは睡眠/ヒーリング系サウンドスケープで$10前後と高水準の推計もある。YPP却下率最高群 |

### 4.3 差別化の実証済みパターン

1. **キャラクター/世界観IP**（Lofi Girl → Lofi Universe）: 完全自社IPはinauthentic判定への最強の防御を兼ねる
2. **機能性の付加**（タイマー、勉強ルーム化）: 「聴く理由」が機能で担保され検索面でも量産系と分離
3. **物語の微付与**（New Bliss: 各動画に短いAmbience Storyline）: 低コストで人間キュレーションの証跡
4. **単一シチュエーション特化**（「Cozy Rainy Bookshop」型）: 汎用より狭い方が有利
5. **B2B/多角化**（Ghelfi: YouTube→Spotify/Patreon/素材販売）: 広告RPM非依存。業務利用者は客単価が高い

### 4.4 日本語圏の空白

- **TRPG環境音**: 国内TRPG人口（CoC等）は大きいのに長尺アンビエンス専門チャンネルが不在（素材サイト中心）
- **ポモドーロ×高品質映像**: 存在するが映像品質が素朴。Abao級の席は空き気味
- **和のシチュエーション**: 囲炉裏・縁側・夜行列車は英語圏に「輸出」できる固有資産
