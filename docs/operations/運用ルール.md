## 1. 目的・適用範囲

- 対象: yancha が生成し **YouTube に公開する全動画**（限定公開・非公開は §5 のゲート対象外だが §2・§3 は常時適用）。
- 目的: 権利・効能表現・品質・スパム/自動化ポリシーの**最終防波堤**を、人間が数分で実行できる形にする。
- 原則: **公開ボタンは人間が押す。** 自動チェックをどれだけ厚くしても、この一線は自動化しない。

## 2. 常時ルール（全動画共通）

| ルール | 内容 | 根拠/実体 |
|---|---|---|
| **AI開示 常時ON** | アップロード時に `containsSyntheticMedia: true`。開示した事実を証跡に残す | [upload ステージ](../../src/stages/upload.ts) |
| **断定的健康効能を謳わない** | 「治る」「不眠が改善」「〇〇Hzで効く」等はNG。「リラックス用」等に限定 | §4 NG辞書 / [policy.ts](../../src/stages/policy.ts) |
| **完全オリジナル・自社ライセンス** | 全アセットのライセンスを保有し証跡を残す。**Civitai派生モデル・CC0素材レイヤーは使わない** | §3 / license.json |
| **生成スタックの優先順** | プロシージャル > Apache2.0/MIT オープンモデル > 商用サービス | 設計 §2 |
| **低品質段階は公開しない** | 品質バー未達は限定公開/非公開で精度を上げる（inauthentic 判定の自衛でもある） | §5 |
| **公開ボタンは人間** | 全自動即公開はしない。レビューは軽量でよいが省略はしない | §5 |

## 3. 証跡保存の必須項目

各動画の `assets/<video_id>/` に**必ず**残す。1つでも欠けたら公開しない（§5-⑦で確認）。

| 証跡 | 何のための証跡か |
|---|---|
| `license.json` | 使用ツール・プラン・利用規約バージョンの記録。**権利主張とAI開示の根拠** |
| seed / 生成ログ | 「機械的コピーではなく自分がパラメータを決めて生成した」ことの再現可能な記録 |
| `uniqueness.json` | 他動画との重複回避（「再利用・繰り返しコンテンツ」判定への反証） |
| `checks.json` | 効能表現リンター・uniqueness・ラウドネスの自動チェック結果（§5-①の入力） |
| 音のステム / 映像パラメータ | 各アセットを自分が生成した工程の証跡（プロシージャル生成の裏付け） |
| `review.md` | 本ゲート（§5）を人間が通した記録。公開判断のログ |

## 4. 効能表現NG辞書（初版）

**この表を「正」とする。** 現行 [policy.ts](../../src/stages/policy.ts) は本辞書の部分実装であり、
将来コードをここへ寄せる（別Issue）。自動リンターはあくまで一次フィルタ。**最終判断は §5-⑥で人間が本表と照合する。**

### 4.1 NG表現 → 言い換え例

| NG（断定的健康効能） | 理由 | 可（言い換え例） |
|---|---|---|
| 不眠が治る / 不眠症が改善 | 医療効果の断定 | 眠る前のリラックスに / 就寝前のひとときに |
| （病気・症状名）が治る・改善する | 医療効果の断定 | ゆったり過ごしたい時間に |
| 自律神経を整える | 医学的作用の断定 | 落ち着いた雰囲気で過ごす |
| 睡眠の質が上がる / 深い睡眠に | 効果の断定 | 静かな夜の環境音として |
| 〇〇Hz で（効く・改善・治る） | 数値による効能の断定 | （周波数を効能の根拠にしない。単に音の説明に留める） |
| ストレスが解消される / ストレスが消える | 効果の断定 | リラックスしたい時間に |
| 免疫が上がる / 血圧が下がる 等の生理指標 | 医療効果の断定 | （使わない） |
| セラピー効果 / ヒーリング効果（効果の断定として） | 効果の断定 | ヒーリング系の環境音 / 癒やしの雰囲気 |
| 医学的に / 科学的に証明された | 権威付けによる効能の断定 | （根拠として使わない） |
| 集中力が上がる（断定） | 効果の断定 | 作業・勉強のおともに / 作業用 |
| 保証 / 必ず / 確実に（効能に係る場合） | 結果の保証 | （使わない） |

### 4.2 運用の原則

- **効能を「約束」しない。用途を「提案」する。**（"治る"ではなく"リラックス用"）
- タイトル・説明・タグ・サムネ文言・チャプター名の**すべて**が対象。
- 迷ったら**より弱い表現**を選ぶ。効能の匂いがしたら消す。

### 4.3 現行 policy.ts パターンとの対応

| policy.ts の正規表現 | 本辞書の該当行 |
|---|---|
| `治(る\|します\|せる)` | 不眠が治る / 病気が治る |
| `改善(する\|します\|できる)` | 不眠症が改善 / 睡眠の質 |
| `病気` | 症状名が治る・改善 |
| `不眠(症)?が(治\|改善)` | 不眠が治る |
| `\d+\s*Hz.*(効く\|効果\|改善\|治)` | 〇〇Hz で効く |

> 本辞書 §4.1 のうち自動リンター未実装の行（自律神経・ストレス・保証 等）は、当面 **§5-⑥の人間目視で担保**する。

## 5. 公開前ゲート手順（人間が数分でやる）

限定公開で生成した動画を**公開に切り替える前**に、人間が上から順に確認する。
自動チェックは厚く、人間は目視数分に絞る設計。**1つでもFAIL/疑義があれば §6 へ。**

- [ ] **① 自動チェック全PASS**: `checks.json`（効能リンター・uniqueness・ラウドネス）が総合 PASS か
- [ ] **② Content IDクレーム**: YouTube Studio で claim の有無を**目視**（APIで取得不可能なため唯一の手段）
- [ ] **③ 品質目視**: 自分が数分見て/聴いて違和感がないか（破綻フレーム・無音・不快なループ継ぎ目 等）
- [ ] **④ 逆画像検索**: 映像キーフレームで既存素材との一致がないか
- [ ] **⑤ AI開示ON確認**: `containsSyntheticMedia: true` で上がっているか（Studio表示も確認）
- [ ] **⑥ 効能表現の最終照合**: タイトル/説明/タグ/サムネ文言を §4 NG辞書と照合
- [ ] **⑦ 権利・証跡**: §3 の必須証跡が `assets/<video_id>/` に揃っているか（license.json 必須）
- [ ] **→ 全て満たしたら、人間が公開ボタンを押す**

> per-video の `review.md`（[review ステージ](../../src/stages/review.ts)出力）は本ゲートの部分実装。
> **本 doc を正とし、review.md のチェック項目は将来ここへ揃える（別Issue）。**

## 6. 逸脱時の扱い

- ①〜⑦のいずれかが **FAIL または疑義** → **公開しない**。限定公開/非公開に戻す。
- 効能表現の疑い（⑥）→ 文言を §4 の言い換えに直して再チェック。
- 品質不足（③）→ 生成パラメータを見直して作り直す（限定公開のまま精度を上げる）。
- 証跡欠損（⑦）→ 該当ステージを再実行して証跡を復元してから再判定。
